## Phase 1 — Сеть и комнаты

**Цель:** два участника подключаются к комнате и слышат друг друга в реальном времени.

**Оценка:** 2–3 недели

### Задачи

- [x] WebRTC signalling-сервер (Node.js + Socket.IO)
- [x] Создание комнаты с уникальным 4-символьным кодом
- [x] Подключение к комнате по коду
- [x] Аудиотранспорт через WebRTC (Opus-кодек через браузер)
- [x] Музыкальные аудио-ограничения (echoCancellation: off, noiseSuppression: off, autoGainControl: off)
- [x] STUN-серверы (Google) для NAT traversal
- [x] TURN-релей (openrelay.metered.ca) для симметричного NAT
- [x] Автопереподключение при обрыве (Socket.IO reconnect + pending rejoin)
- [x] Изоляция комнат — участники слышат только свою комнату
- [x] Ограничение участников по комнате (rate limiting на события)
- [ ] Пароль на комнату (приватная комната)
- [ ] Лимит участников задаётся хостом (рекомендуемый максимум: 8)
- [ ] Индикатор пинга каждого участника в реальном времени
- [ ] Подключение по прямой ссылке (invite link)
- [ ] Сохранение настроек посещённых комнат (паттерны, конфигурация)
- [ ] История комнат на стартовом экране

### Критерий готовности

- [x] Два участника подключаются и слышат друг друга
- [ ] Пинг отображается у каждого участника
- [ ] Приватная комната с паролем работает

## Phase 2 — Миксер и запись

**Цель:** участник записывает инструмент, аудио попадает на таймлайн без сдвига.

**Оценка:** 3–4 недели

### Задачи

**Аудиодвижок:**
- [ ] Определение аудиодрайвера: ASIO (Windows) / CoreAudio (macOS) / ALSA (Linux)
- [ ] Выбор аудиоустройства ввода/вывода в настройках
- [ ] Настройка размера буфера (buffer size)
- [ ] Минимальная задержка через нативный аудиодрайвер

**Миксер (частично реализован):**
- [x] Input-каналы для каждого участника
- [x] Регулировка громкости (fader) на каждом канале
- [x] Кнопка Mute на каждом канале
- [x] Кнопка Solo на каждом канале
- [x] VU-метр (индикатор уровня сигнала) на каждом канале
- [x] Стерео панорама на каждом канале
- [x] Мастер-шина с компрессором
- [ ] Кнопка Record на каждом канале
- [ ] Мониторинг латентности аудио на каждом канале
- [ ] Send/return шина для общих эффектов комнаты
- [ ] VST/AU хост: загрузка плагинов на input-канал
- [ ] Цепочка VST-плагинов на канале
- [ ] Копирование конфигурации VST на дорожку при записи

**Запись:**
- [ ] Запись pipeline: preroll → чистый WAV-поток
- [ ] Preroll длительностью из настроек метронома
- [ ] Немедленный доступ к записанному аудио в WAV
- [ ] Фоновое кодирование в MP3 (не блокирует интерфейс)
- [ ] Автоматическая компенсация латентности (latency compensation)

### Критерий готовности

- [ ] Участник записывает инструмент с обработкой
- [ ] Аудио попадает на таймлайн без временного сдвига
- [ ] MP3 кодируется в фоне без зависания UI

## Phase 3 — Монтажный стол и MIDI

**Цель:** записать, отредактировать и экспортировать полноценный трек.

**Оценка:** 3–4 недели

### Задачи

**Arrange (Timeline):**
- [ ] Окно монтажного стола (Arrange window)
- [ ] Аудио-дорожки (WAV / MP3)
- [ ] MIDI-дорожки
- [ ] Дорожки автоматизации
- [ ] Временная шкала с зумом и скроллом
- [ ] Drag-and-drop клипов
- [ ] Trim (обрезка клипов)
- [ ] Split (разрезание клипов)
- [ ] Undo / Redo
- [ ] Snap to grid (привязка к сетке)

**Piano Roll:**
- [ ] Открытие MIDI-дорожки в piano roll
- [ ] Редактирование нот (добавление, удаление, перемещение)
- [ ] Velocity на каждой ноте
- [ ] Квантизация
- [ ] Редактор по стандарту Reaper / Cubase / Logic

**MIDI:**
- [ ] WebMIDI API: приём сигналов с внешних устройств
- [ ] Нативный MIDI bridge
- [ ] MIDI routing на дорожки

**Экспорт:**
- [ ] Финальный mixdown в WAV
- [ ] Финальный mixdown в MP3
- [ ] Экспорт проекта

### Критерий готовности

- [ ] Записанный аудио-клип появляется на таймлайне
- [ ] MIDI-дорожку можно редактировать в piano roll
- [ ] Проект экспортируется в WAV/MP3

## Phase 4 — Метроном и драм-машина

**Цель:** все участники слышат метроном и драм-машину синхронно, паттерн экспортируется в MIDI.

**Оценка:** 2 недели

### Задачи

**Метроном:**
- [x] BPM-контроль (60–240)
- [ ] Размер такта (4/4, 3/4, 6/8 и др.)
- [ ] Выбор сильной доли
- [ ] Длительность preroll (в тактах) в настройках
- [ ] Визуальный клик (мигание интерфейса в такт)
- [ ] Режим только sync (звук метронома отключён, сигнал для DAW сохранён)

**Драм-машина (частично реализована):**
- [x] Step sequencer — 4 канала: kick, snare, hat, crash
- [x] 16 шагов по умолчанию
- [x] Встроенные сэмплы (kick, snare, hat, crash)
- [x] Паттерны синхронизированы с BPM комнаты
- [x] Синхронизация паттернов между участниками (LWW)
- [ ] Варианты шагов: 8 / 16 / 32
- [ ] Velocity на каждом шаге
- [ ] Swing / groove параметр
- [ ] Несколько паттернов с переключением
- [ ] Chaining паттернов
- [ ] Экспорт паттерна → MIDI-дорожка на монтажном столе

**BPM синхронизация (частично):**
- [x] Единый BPM для всей комнаты (управляет хост)
- [x] BPM синхронизирован с драм-машиной и транспортом
- [ ] NTP-подобная синхронизация тактовой сетки (clock sync)
- [ ] Коррекция дрейфа (drift correction)

### Критерий готовности

- [x] Драм-машина работает и синхронизируется
- [ ] Все участники слышат метроном синхронно
- [ ] Паттерн экспортируется в MIDI-дорожку

## Phase 5 — UI и полировка

**Цель:** полный сценарий «репетиция от открытия до экспорта» без ошибок.

**Оценка:** 2–3 недели

### Задачи

**Стартовый экран:**
- [x] Выбор: создать комнату / присоединиться
- [ ] История посещённых комнат
- [ ] Открыть сохранённую комнату

**Тулбар:**
- [x] BPM-переключатель (общий для комнаты)
- [x] Запуск / остановка транспорта
- [ ] Отдельная кнопка запуска/остановки метронома
- [ ] Кнопка настроек метронома
- [ ] Кнопка вызова монтажного стола
- [ ] Кнопка вызова драм-машины

**Чат комнаты:**
- [ ] Текстовый мессенджер внутри сессии
- [ ] История сообщений на время сессии

**Права участников:**
- [ ] Хост может mute любого участника
- [ ] Хост может kick (выгнать) участника
- [ ] Значки ролей (хост / гость)

**Настройки:**
- [ ] Выбор формата записи (WAV / MP3)
- [ ] Выбор аудиоустройства ввода/вывода
- [ ] Настройка размера буфера

**Стабильность:**
- [ ] Автосохранение состояния комнаты
- [ ] Защита от потери данных при крэше
- [ ] Нагрузочное тестирование: комнаты на 4–8 участников

**Визуальный дизайн (реализован):**
- [x] Тёмная тема (Persian Luxury Cyber: чёрный / графит / золото / кристалл)
- [x] Design tokens (CSS переменные)
- [x] Адаптивная вёрстка
- [x] Активный говорящий (gold glow на тайле)

### Критерий готовности

- [ ] Полный сценарий: открыть → создать комнату → репетиция → запись → экспорт
- [ ] Нет потери данных при обрыве и переподключении
- [ ] Комната с 4 участниками стабильна
